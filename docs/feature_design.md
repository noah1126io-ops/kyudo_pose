# 弓道射型評価用 特徴量設計書 v0.1

## 目的

本設計書は、弓道動画から射型評価を行うために必要な特徴量を定義する。

ここでいう特徴量とは、姿勢推定や物体認識の結果から計算される数値指標を指す。
将来の用途は以下の 3 つである。

1. ルールベース評価
2. 教師あり学習用のモデル入力
3. モバイルアプリ上での評価表示

## 前提

本設計書では、横動画・単一人物・1 射単位を前提とする。

ただし弓道評価では、人の関節だけでは不十分であり、以下の 3 系統を扱う。

1. 人体特徴量
2. 弓具特徴量
3. 基準線特徴量

## 設計方針

- 最初は 2D 横動画で安定して取れる特徴量を優先する
- ルールベースで意味が明確な特徴量から採用する
- 後から学習モデルに投入できるよう、フレーム単位時系列で保持する
- 生の座標だけでなく、角度・距離・速度・安定性も定義する
- 弓道特有の重要観点として、矢線、弓の傾き、弭の軌道を独立して扱う

## 特徴量の大分類

### A. 人体特徴量

MMPose / RTMPose の 2D キーポイントから得る特徴量。

### B. 弓具特徴量

弓、矢、弦、本弭、末弭などの位置・傾き・相対関係を表す特徴量。

### C. 基準線特徴量

地面、鉛直、撮影基準線など、評価基準となる外部基準。

### D. 時系列特徴量

フレーム推移から得る速度、加速度、安定性、フェーズ長など。

## 5. 特徴量の認識手段

ユーザー指摘の通り、人体だけでは足りない。認識手段を分ける必要がある。

### 5-1. 人体

- RTMPose / MMPose

### 5-2. 弓・矢・弦・弭

候補:

- 専用キーポイント検出
- 線分検出
- セグメンテーション
- 物体検出 + 端点推定

初期段階では次の優先順を推奨する。

1. 矢線の線分検出
2. 弓の端点検出
3. 本弭・末弭の点検出
4. 必要なら弦検出

### 5-3. 地面

候補:

- 手動キャリブレーション
- 画面内の床線の線分検出
- 既知の撮影条件による固定補正

最初は撮影条件を固定し、地面線は手動または準手動で与えるのが現実的。

## 6. 初期実装の優先順位

### 最優先

- 左右肩高さ差
- 胴体傾き
- 左右肘角度
- 左右手首高さ
- 会の静止度
- 矢線角度
- 矢線安定性

### 次点

- 弓全体の傾き
- 本弭・末弭の軌道
- 地面基準補正後の角度
- フェーズ長

### 後段

- 弦の挙動
- 細かな手の内評価
- 3D 補正

## 7. 特徴量の正規化

撮影距離や体格差の影響を減らすため、以下で正規化する。

- 肩幅
- 体幹長
- 動画 fps
- 地面基準

例:

- `left_wrist_y_normalized = left_wrist_y / torso_length_px`
- `motohazu_lateral_sway_normalized = motohazu_lateral_sway / shoulder_width_px`

## 9. 注意点

- 人体キーポイントだけでは弓道評価は不十分
- 矢線、弓、弭、地面は別系統の認識対象として扱うべき
- 初期段階では、弓具と地面は完全自動より半自動でもよい
- 最終的なスマホ実装では、軽量な線分検出や小型検出器が必要になる可能性が高い
